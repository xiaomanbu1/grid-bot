# 多币种 / 多交易所使用说明

## 切换币种 (三种方式)

1. **Telegram 菜单**: 发 `/menu` → 点 "🪙 切换币种" → 选 BTC/ETH/SOL/BNB/XRP/DOGE 快捷按钮
2. **Telegram 文字**: 发 `/symbol DOGE` 切到任意币种(只要交易所有这个永续)
3. **配置文件**: 改 `config.local.yaml` 的 `perp_symbol`, 然后 `docker-compose restart`

切换币种时会自动: 撤掉旧币种所有挂单 → 按新币种近30天波动率自动调格距和范围 → 重新锚定。

## 自动调参逻辑

不同币波动差异大, 用同一套参数会出问题(BTC的密格距用在DOGE上会被手续费吃光)。所以切币时自动:

- 格距 ≈ 日波动率 × 40%, 限制在 0.5%~3%
- 范围 ≈ 日波动率 × 8, 限制在 ±8%~±30%

例: BTC 日波动~1.5% → 格距0.52% 范围±10%; DOGE 日波动~6% → 格距2.1% 范围±30%。

## 切换交易所 (币安 ↔ Gate)

改 `config.local.yaml` 的 `exchange.id`:

- `binance`: 主流币 + 币安上的中小币 (几百个)。API规范, 推荐
- `gate`: 隐私币 (XMR/ZEC等币安没有的)。注意 Gate 统一账户的 API 权限问题

**注意**: 隐私币(XMR等)币安已下架, 必须用 gate。换交易所要换成对应所的 API key, 然后 docker-compose restart。

## 各币种资金门槛 (币安永续最小下单量)

不同币最小下单量不同, 影响单格最小金额:
- BTC: 0.001 BTC ≈ 60U
- ETH: 0.01 ETH 左右
- SOL/DOGE 等: 最小量小, 单格门槛低, 资金少更友好

切到新币后看启动日志 `市场: XXX 最小下单量=...`, 据此调 usdt_per_grid。

## 风险提醒

小币/隐私币的风险远高于 BTC:
- 流动性差, 网格滑点大
- 容易被交易所下架(XMR就是例子)
- 波动剧烈, 即使自动调宽了格距, 单边行情仍可能快速触发熔断
- 山寨币归零风险, 别把大仓位长期放在小币网格上

主流币(BTC/ETH)相对最稳, 适合长期跑网格。
